# 코인 투자 감정 분석 게임 — 기획서

> **해커톤 프로젝트** · 대주제: **감정(Emotion)**
> Upbit API 필수 사용 · 출력 형태 자유

---

## 0. 한 줄 정의

과거 실제 코인 시장에서, **차트만 보고 내린 결정**과 **군중의 감정을 본 뒤 바꾼 결정**을 비교해
**"나는 시장의 감정에 얼마나 휩쓸리는 사람인가"**를 진단하는 거울.

이건 "투자 잘하는 게임"이 아니다. 수익률은 결과가 아니라 **감정에 휩쓸린 대가**를 보여주는 장치다.

---

## 1. 후킹 메시지

시작 화면 카피:

> **"만약 내가 코인 투자를 한다면 성공할 수 있을까?"**

코인 입문자 / 관망자의 호기심을 유도하는 진입점.

---

## 2. 핵심 메커니즘 (게임의 심장)

매 턴 **2단계 노출** 구조로 감정에 흔들린 정도를 측정한다.

```
① 차트 공개        → 현재 턴까지의 시세만 (이성의 닻)
② 1차 선택         → 매수 / 매도 / 보유 (직감, 기록만 · 체결 안 함)
③ 감정 신호 공개    → 공포탐욕지수 → 커뮤니티 → 뉴스 순
④ 2차 선택         → 매수 / 매도 / 보유 (최종 · 실제 체결)
```

**①(이성) vs ④(감정 노출 후)의 차이 = 플레이어가 감정에 흔들린 양.**
이 격차가 게임의 모든 분석의 핵심 데이터다.

> 미래 구간 차트는 절대 보이면 안 됨. **현재 턴까지만** 공개.

---

## 3. 전체 게임 흐름

```
게임 시작
   ↓
4개 시나리오 중 1개 선택 (2×2 그리드 · 해시태그 설명)
   ↓
(선택) 투자 성향 사전 테스트 3~5문항
   ↓
코인 2~3개 + 시드머니 1억원 지급
   ↓
[매 턴 반복 · 총 12턴 고정]
   ├ ① 차트 공개 (현재 턴까지)
   ├ ② 1차 선택 (직감 · 기록만)
   ├ ③ 감정 신호 공개 (공탐지수 → 커뮤니티 → 뉴스)
   └ ④ 2차 선택 (최종 · 체결)
   ↓
12턴 종료
   ↓
감정 분석 결과 (① vs ④ 차이 = 휩쓸린 양)
   ↓
감정 유형 진단 + 실제 시장 결과 공개 + 개선법
```

- 턴 넘기기(방관) 가능
- 한 턴 안에서는 시간이 흐르지 않음
- 코인 수는 2~3개 (많아지면 감정 게임이 아니라 포트폴리오 관리 게임이 됨)

---

## 4. 시나리오 (4개)

**턴 수 12 고정, 간격만 시나리오마다 변동** (분석 비교를 위해 턴 수 고정)

| # | 시나리오 | 특징 | 턴 간격 | 해시태그 예시 |
|---|----------|------|---------|---------------|
| 1 | 21년 도지코인 | 큰 등락·고변동성 | 3일/턴 | #밈코인 #롤러코스터 |
| 2 | 21년 코인 고점 | 상승 → 고점 | 1주/턴 | #불장 #천장은어디 |
| 3 | 22년 거래소 파산 | 폭락장 | 1주/턴 | #패닉 #뱅크런 |
| 4 | 25년 하반기 | 최근장 | 1주/턴 | #최근장 #실전 |

- 각 시나리오마다 그 시기 **영향력 컸던 코인 2~3개** 선정
- 선택 화면: 2×2 그리드, 해시태그로 짧게 분위기 전달

## 5. 데이터 출처

| 데이터 | 출처 | 비고 |
|--------|------|------|
| 차트 / 시세 | **Upbit 캔들 API** | 과거 데이터, 페이지네이션 필요(200개 제한) |
| 공포탐욕지수 | 외부 소스 (예: Alternative.me) | Upbit엔 없음 |
| 뉴스 | 사전 수집 / 작성 | Upbit엔 없음 |
| 커뮤니티 글 | **사전 작성** | 실시간 생성은 불안정, 미리 제작 |

### Upbit API 사용처
- **유일 핵심 용처 = 과거 캔들(시세) 조회**
- 차트 렌더링 + 매매 체결가 + 수익률 계산이 전부 Upbit 데이터 위에서 동작
- `to` 파라미터로 과거 시점 조회 / 한 번에 최대 200개 → 페이지네이션
- ⚠️ **21·22년 데이터가 실제로 조회되는지 사전 검증 필수**

### 커뮤니티 글 분량
- 4 시나리오 × 12턴 ≈ **48개** 미리 작성
- 톤: 날것의 감정 — "가즈아", "존버", "물렸다", "지금이라도 타야하나"

## 6. 분석 지표 (감정 언어로 표현)

| 지표 | 측정 | 결과 멘트 예시 |
|------|------|----------------|
| FOMO | 고점에서 추격매수 비율 | "군중이 환호할 때 따라 들어갔습니다" |
| 패닉셀 | 하락장 성급한 매도 비율 | "공포가 퍼지자 가장 먼저 손을 놓았습니다" |
| 정보 의존도 | 공탐/커뮤니티/뉴스 중 최대 영향원 | "당신을 가장 많이 흔든 건 커뮤니티의 분노였습니다" |
| 분산투자 | 자산 분배 적절성 | (보조 지표) |
| **1차 vs 2차** | 두 선택의 격차 | **"차트를 믿었지만, 공포 앞에서 N번 마음을 바꿨습니다"** ← 결론 |

> ※ 다이아몬드 핸즈 지수는 제외

### 실제 시장 비교 멘트 예시
- "실제 최고점보다 2턴 빨리 매도했습니다."
- "공포지수 20 이하일 때 평균보다 18% 높은 비중으로 매수했습니다."

---

## 7. 결과 화면

1. 전체 기간 차트 + **매수/매도 시점 마커** 표시
2. 감정 유형 이름 + 비슷한 투자자 유형
3. **실제 시장에서는 어떤 결과였는지 공개**
4. 유형별 행동 개선 방법

---

## 8. 차별점

다른 투자 시뮬과 달리, 정량 데이터로 수익을 내는 게임이 아니라
**"감정 반응 패턴을 거울처럼 보여주는 진단 도구."**

`차트 → 직감 입력 → 감정 정보 노출 → 최종 결정`의 순서가 그 진단을 가능하게 하는 핵심 장치.

## 10. 미결정 사항

- [ ] **사전 성향 테스트(3~5문항) 유지 여부**
  - 빼도 1차/2차 차이로 진단 가능
  - 넣으면 "내가 생각하는 나 vs 실제 행동" 대비가 생겨 결과가 더 극적

## 11. 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| Frontend | **Next.js 14** (App Router) | 해커톤 속도, SSR로 API 키 보호, Vercel 원클릭 배포 |
| 스타일링 | **Tailwind CSS** + **shadcn/ui** | 빠른 UI 구성, 다크 테마 기본 지원 |
| 차트 라이브러리 | **Lightweight Charts** (TradingView) | 캔들스틱 특화, 가볍고 빠름, 무료 |
| 상태 관리 | **Zustand** | 게임 상태(턴·잔고·선택 기록) 전역 관리, 보일러플레이트 최소 |
| 데이터 페칭 | **Axios** | Upbit REST API 호출 |
| Upbit API 연동 | **Next.js API Routes** 경유 | 브라우저 직접 호출 시 CORS 문제 우회 |
| 배포 | **Vercel** | Next.js 최적화, 무료 플랜으로 충분 |

### Upbit API 핵심 엔드포인트

```
GET https://api.upbit.com/v1/candles/days
  ?market=KRW-BTC
  &to=2021-04-14T00:00:00Z   ← 과거 시점 지정
  &count=200                  ← 최대 200개
```

- 21·22년 데이터 조회 가능 여부 **사전 검증 필수**
- 200개 제한 → 턴 간격에 맞게 페이지네이션으로 분할 수집
